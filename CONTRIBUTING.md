# Contributing to Forge

Hi there! We're thrilled that you'd like to contribute to **Forge**. Contributions to this project are [released](https://help.github.com/articles/github-terms-of-service/#6-contributions-under-repository-license) to the public under the [project's open source license](LICENSE).

Please note that this project is released with a [Contributor Code of Conduct](CODE_OF_CONDUCT.md). By participating in this project you agree to abide by its terms.

## 1. Governance & Rules
Before contributing, please read **[AGENTS.md](AGENTS.md)** ("The Constitution"). It defines:
- **Core Principles**: Library-First, TDD, Documentation-Driven.
- **Workflow**: RDD -> DDD -> TDD -> CDD.
- **Documentation Standards**: How to structure `docs/`.

## 2. Prerequisites

1. Install [Python 3.11+](https://www.python.org/downloads/)
2. Install [uv](https://docs.astral.sh/uv/) (optional but recommended)
3. Install [Git](https://git-scm.com/downloads)

## 3. Development Workflow

1. **Fork and Clone**:
   ```bash
   git clone https://github.com/your-username/forge.git
   cd forge
   ```

2. **Install Dependencies**:
   ```bash
   pip install -e .
   pip install pytest pytest-mock
   ```

3. **Verify Installation**:
   ```bash
   forge --help
   ```

4. **Running Tests**:
   ```bash
   pytest tests/
   ```

## 4. Submitting a Pull Request

1. **Create a Branch**: `git checkout -b feat/my-feature`
2. **Follow the Methodology**:
   - Update `docs/01-plans/` if changing strategy.
   - Update `docs/02-tasks/` to track your work.
   - Write tests (`tests/`).
   - Implement code (`src/forge/`).
3. **Commit**: Write descriptive commit messages.
4. **Push and PR**.

## 5. AI Contributions

> [!IMPORTANT]
> If you are using **AI assistance** (ChatGPT, Claude, Agents), please disclose it in the PR description.

Example:
> "This PR was generated by Claude Code following the instructions in `AGENTS.md`."
